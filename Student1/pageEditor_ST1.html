Splash Page

Technical Description

The Loading Screen is a transitional splash page that presents a fullscreen background video and animated content before redirecting users to the home page after a delay. It leverages semantic HTML elements and advanced CSS animations for an engaging user experience. The background video (`<video>`) provides immersive visual storytelling, while animated text and a three-dot spinner create visual interest during the wait. CSS keyframes manage typing, blinking, fading, and bouncing effects. A `meta` tag in the `` automatically redirects the user after 6 seconds.

Key Tags and Techniques

  • <video> — Embeds the fullscreen background video.
  • <meta http-equiv="refresh"> — Automatically redirects to the homepage.
  • <img> — Displays the site logo with a fade-in effect.
  • keyframes — Powers animations such as typing, blinking, bounce, and fade-in.
  • object-fit: cover — Ensures the video fills the entire screen.
  • z-index — Manages layering between video and content.
  • position: fixed — Keeps the loader and video elements in place.
  • animation-delay — Staggers the appearance of content for a polished effect.
  • Flexbox — Centers loader content vertically and horizontally.
  • transition — Smoothens changes in opacity and visibility.

Accessibility

  • Readable and responsive typography with high contrast.
  • Alternative text on the logo for screen readers.
  • Graceful fallback text in case the video format isn't supported.
  • Logical content structure improves screen reader navigation.
  • Motion kept minimal and non-intrusive for comfort.

Volunteer Page

Technical Description

The Volunteer Page presents a catalog of conservation programs using dynamic and responsive card components. Each project is showcased in a visually organized card layout within a Flexbox container, adapting across screen sizes. Users can sort programs using a dropdown menu. Cards include images, location-specific project titles, tags, durations, and pricing. Hover effects animate the cards for enhanced interactivity, and calls to action are provided through "View Project" and "Apply" buttons. A feedback section allows users to share their reviews using emoji-based ratings and a text input area. Verified participant reviews are displayed using styled testimonial cards.

Key Tags and Techniques

  • <div class="card"> — Container for each volunteer program.
  • <img src="..." alt="..."> — Displays project images with descriptions for accessibility.
  • <select> and <option> — Enables sorting functionality for filtering programs.
  • <div class="tags"> — Highlights features such as duration and cost using tag elements.
  • Flexbox — Arranges cards and content responsively using flex-wrap and gap.
  • box-shadow — Adds depth to project and review cards.
  • hover effects — Adds smooth interactivity to buttons and cards.
  • object-fit: cover — Maintains aspect ratio of images without distortion.
  • border-radius — Smooths edges of cards, inputs, and buttons.
  • transition — Animates scale and hover effects smoothly.

Accessibility

  • Alt text for all images for screen readers.
  • Color contrast maintained for readability.
  • Keyboard-navigable dropdown and form inputs.
  • Hover and focus effects enhance usability across devices.
  • Feedback form includes clearly labeled inputs and emoji symbols for expressive input.

Content Page

Technical Description

The Mountain Ecosystem content page is designed using semantic HTML and a responsive layout to present educational information clearly. It utilizes <main> and <aside> to separate main content and quick navigation. Each topic section is wrapped in a <div class="card">, styled with spacing and imagery to enhance engagement. The design uses Flexbox to support responsive side-by-side layout between the sidebar and main content. Smooth scrolling and a scroll-to-top button improve usability on long pages.

Key Tags and Techniques

  • <main>, <aside> — Structure the page layout semantically.
  • <div class="card"> — Used to wrap individual topic sections.
  • <h2>, <p>, <img> — Present structured content with visuals.
  • Sidebar Navigation — Provides quick links to internal page sections.
  • Scroll-to-top Button — Enhances navigation usability.
  • Responsive Design — Ensures proper layout on desktops, tablets, and mobiles.
  • Padding and margin — Improve visual spacing and readability.

Accessibility

  • Alt text provided for all images describing mountain features and wildlife.
  • Headings follow a logical hierarchy (<h2>, <h3>).
  • Sidebar enhances keyboard navigation with internal links.
  • Content does not overflow on small screens.
  • Good contrast between text and background.

Challenges and Lessons Learned

While developing the Team, Volunteer, and Mountain Ecosystem pages, several challenges emerged that tested both technical skills and design understanding. One of the major challenges was ensuring responsiveness across various screen sizes. Making sure that elements like profile cards, sidebars, and content blocks adapted gracefully to mobile and tablet devices required meticulous use of Flexbox, media queries, and careful layout planning. Styling consistency was another hurdle, particularly due to the use of multiple external stylesheets. Conflicts between shared classes and overlapping CSS rules demanded better modularization and more descriptive class naming conventions. Handling images such as profile photos and banner visuals also presented difficulty. Ensuring they retained their aspect ratio, remained sharp, and integrated well into the layout without distortion required the use of object-fit: cover, fixed dimensions, and responsive design techniques. Creating intuitive dropdown menus for the navbar, without relying on JavaScript, called for precise CSS handling to maintain a smooth hover interaction and layout stability. Accessibility was an ongoing focus, with attention given to semantic HTML, logical heading structure, and meaningful alt text for images to support screen readers and ensure usability for all users.

Compliance

The development of the Team Page, Volunteer Page, and Mountain Ecosystem Content Page was carried out in adherence to JANET regulations and recognized web development standards. All HTML and CSS code was validated to ensure semantic correctness and cross-browser compatibility. Accessibility requirements were met by incorporating meaningful alt attributes, logical heading structures, keyboard-navigable elements, and respecting user preferences for reduced motion. Responsive design techniques, such as Flexbox and media queries, ensured optimal usability across various devices. Visual consistency was maintained through a unified design language, while performance was optimized using compressed images and modular CSS. By aligning with JANET guidelines and W3C standards, the pages meet institutional compliance for quality, accessibility, and secure content delivery in educational environments.

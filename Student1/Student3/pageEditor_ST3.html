Feedback Form Page

Technical Description

The Feedback Form was designed using semantic HTML elements such as <form>, <label>, <input>, <textarea>, and <select> . This ensures proper structure and allows screen readers to interpret the form correctly.CSS Flexbox was applied to center the form on the page, making the layout responsive across different screen sizes. Visual styling includes the use of border-radius to create rounded input fields, box-shadow for subtle depth , and consistent padding for spacing. A JavaScript character counter provides live feedback within the comment field to enhance user experience.

Key Tags and Techniques

  • <form> — Defines the form structure.
  • <label for="id"> — Connects labels to form fields for accessibility.
  • <input type="text/email/number"> — Gathers user inputs.
  • <textarea> — Allows multiline user comments.
  • <select> with <option> — Dropdown selection for user preference.
  • required attribute — Ensures fields are mandatory.
  • max-width, padding — Controls layout width and spacing.
  • border-radius — Smooth, rounded corners.
  • box-shadow — Adds visual depth.
  • Flexbox — Aligns and centers the form.

Accessibility

  • Clear labels linked to inputs via for attribute.
  • Logical tab order for keyboard users.
  • High-contrast text and input borders.
  • Visible hover and focus states for interactive elements.
  • Real-time character counter for clarity.

Team Page

Technical Description

The Team Page showcases group members using visually appealing profile cards. Each member's information is displayed within a <div> styled with Flexbox to ensure even spacing and responsiveness. Profile images are contained within <img> tags, styled with border-radius for circular presentation and object-fit to maintain aspect ratio. Hover effects enhance interactivity, and box-shadow creates a subtle 3D effect around each card. The layout adjusts automatically across devices using flexible sizing.

Key Tags and Techniques

  • <div class="member"> — Profile card container.
  • <img src="..." alt="..."> — Displays and describes profile images.
  • <h3> — Displays member names and roles.
  • Flexbox — Aligns cards using display: flex; flex-wrap: wrap; justify-content: center.
  • border-radius: 50% — Creates circular profile images.
  • object-fit: cover — Prevents image distortion.
  • box-shadow — Adds depth to cards.
  • gap — Maintains spacing between members.

Accessibility

  • Meaningful alt text on all images.
  • Keyboard focus styles for accessibility.
  • Hover effects respect reduced motion preferences.
  • Responsive design prevents overlapping content.

Content Page (ST3)

Technical Description

The Content Page delivers structured information using semantic tags like <section>, <article>, and <h2>. Lists are presented with <ul> and <li>, styled with spacing to improve readability. A background gradient provides visual appeal without sacrificing contrast. CSS properties such as padding, margin, and max-width enhance content readability. The layout is fully responsive, adapting gracefully across various screen sizes.

Key Tags and Techniques

  • <section>, <article> — Divides content into meaningful blocks.
  • <h2>, <h3> — Establish heading hierarchy.
  • <ul>, <li> — Organized lists for key information.
  • CSS gradient background — Visual enhancement using:
    background: linear-gradient(135deg, #d7f5d3 25%, #a3d89f 75%);
  • padding, margin — Controls spacing.
  • Responsive design — Ensures mobile compatibility.

Accessibility

  • Logical heading structure for screen readers.
  • Adequate spacing improves readability.
  • High-contrast text over background.
  • Images include descriptive alt attributes.
  • Layout avoids content overflow on small screens.

Challenges and Lessons Learned

A major challenge in developing the Feedback Form was achieving a responsive, centered design. Initially, the form appeared unaligned on smaller screens. By applying CSS Flexbox and max-width properties, I ensured proper alignment across devices.

The Team Page also presented difficulties with maintaining uniform image sizes. Using fixed width, height, and object-fit properties resolved the issue. This project enhanced my skills in responsive design, accessibility, and form validation.

Compliance

The pages follow Janet web publishing regulations by prioritizing accessibility, responsive design, and valid code. All images include alt text, headings follow a logical structure, and color contrast meets accessibility standards. The pages were validated using W3C tools, ensuring correct HTML and CSS syntax.
